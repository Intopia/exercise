Complex table examples

Complex tables can be difficult for people using assistive technologies to understand and navigate.

As the number of headers associated with each data cell increases, users must listen to, understand and remember more contextual information while moving through the table.

Some assistive technologies may also present complex header relationships differently, making the experience less consistent. Wherever possible, simplify tables so that each data cell is associated with no more than two headers.

Example 2: Simplified table

This table is considered simple because each data cell is associated with only a column header and a row header. This reduces the amount of contextual information users need to understand and remember while navigating the table.

The same information is presented in a simpler structure by repeating some values rather than grouping them under additional headers. This reduces the number of headers associated with each data cell.
