Testing accessible names

Links and buttons using content

The contents can be used to generate the accessible name for the <a> and <button> elements:

More 
<a href="#">
  More
</a>

<button>
  Close
</button>

Links and buttons using hidden content

Hidden content can be used to generate a more detailed accessible name:

More about fishes 
<a href="#">
  More
  <span class="hidden">about fishes</span>
</a>

<button>
  Close
  <span class="hidden">modal</span>
</button>

Links and buttons using aria-label

The aria-label attribute can be used to create an accessible name that is different to the visible text label:

Next 
<a href="#" aria-label="Next article">
  Next
</a>

<button aria-label="Dismiss warning notification">
  Dismiss
</button>

Links and buttons using title

What about the title attribute? It may not create an accessible name as it is the weakest method. More importantly, this is a bad practice as the title is not fully accessible.

Find out more 
<a href="#" title="Find out more about trees">
  Find out more
</a>

<button title="Exit">
  <img src="cross.png" alt="">
</button>

Inputs using the <label>

The contents of the <label> attribute can be used to generate the accessible name for the <input>:

<label for="address">Address</label>
<input id="address" type="text">

Inputs using the aria-label

The contents of the aria-label attribute can be used to generate the accessible name for the <input>:

<input type="text" aria-label="Search the site">

Inputs using the title

The contents of the title attribute can be used to generate the accessible name for the <input>. As mentioned above, this is a bad practice as the title is not fully accessible.

<input type="text" title="Bank details">

Inputs using the placeholder

The contents of the placeholder attribute can be used to generate the accessible name for the <input>. This is also a bad practice as the placeholder is not fully accessible.

<input type="text" placeholder="MM/DD/YYYY">
