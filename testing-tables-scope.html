Testing the table scope attribute

This page explores whether the scope attribute makes any difference when used in a simple table. The two tables below contain identical data and structure. The only difference is that the first table uses the scope attribute and the second does not.

As you test each table, listen to what is announced for each data cell and compare the results.

Example 1: using scope

Greensprings Running Club Personal Bests
Name 1 mile 5 km 10 km
Mary 8:32 28:04 1:01:16
Betsy 7:43 26:47 55:38
Matt 7:55 27:29 57:04
Todd 7:01 24:21 50:35

Example 2: without scope

Greensprings Running Club Personal Bests
Name 1 mile 5 km 10 km
Mary 8:32 28:04 1:01:16
Betsy 7:43 26:47 55:38
Matt 7:55 27:29 57:04
Todd 7:01 24:21 50:35

What we found

In our testing, the scope attribute made no observable difference to how modern screen readers announced this simple table. The row and column headers were correctly associated with each data cell in both examples.

This doesn't mean the scope attribute is never useful. However, for simple tables like these, modern browsers and assistive technologies generally infer the relationships between header and data cells without it.
